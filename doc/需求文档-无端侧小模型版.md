# 学习指认AI产品 — 功能性需求文档（无端侧小模型版）

## 零、文档说明（与主需求文档的关系）

- **主文档**：`doc/需求文档.md`，为产品功能全集基线。
- **本文档（分身）**：在**不依赖学生端 / 家长端设备上运行的神经网络小模型**（如 MediaPipe Hands、端侧轻量 OCR、端侧神经网络 VAD 等）的前提下，对交互与能力做**等价替代描述**；**云侧大模型与云服务**（DeepSeek、火山 STT、腾讯云 TTS、声网等）仍为能力主体。
- **仍可使用**：操作系统能力（相机预览、麦克风采集、硬件编解码）、**Web Speech API / 系统语音识别与合成**（作为系统服务，不视为自研端侧小模型）、简单**能量阈值静音判断**（非学习式模型）。
- **家长端**：与主文档一致，**第一阶段优先 Android 手机 APK**；Web 后续。
- **冲突处理**：与主文档功能目标一致；仅「实现路径」不同处以**本文档**为准（无端侧小模型路径）。

## 一、文档概述

### 1.1 项目背景

本项目针对K12阶段学生课业辅导、错题巩固的核心需求，打造学生端（iPad）+ 家长端（**优先 Android 手机**，Web 后续）双端协同的学习指认AI产品。学生端以**语音对话**为核心交互方式，结合USB外接摄像头实现试题实时投屏与指认，通过AI完成试题识别、批改、讲解、错题生成。家长端实现远程查看、权限管控、错题审批、远程介入教学与协同指导。双端通过同一账号登录实现数据同步，所有数据统一存储至云端。

### 1.2 核心目标

- 以**语音实时对话**为主要交互方式，学生与AI进行自然语言交流，无需手动输入
- AI具备情境感知能力，根据对话内容自动切换聊天模式与作业辅导模式
- 通过USB摄像头实现试题实时投屏、OCR识别、**题目指认交互**（点选/框选/题号 + 语音，无端侧手指模型）
- AI遮挡检测：当学生遮挡摄像头画面时，AI主动提醒
- 搭建家长端远程管控体系，支持远程介入教学、画中画交互、权限控制
- 所有OCR识别结果按学生年级、科目、章节自动归类存储至云端
- 家长端可按科目/时间/章节创建错题集，AI自动生成带审核机制的练习试题
- AI可命名、可选声音和讲话节奏，实现个性化交互体验

### 1.3 适用范围

本文档适用于产品研发、设计、开发、测试全流程人员，明确产品功能性需求。

### 1.4 双端定位

- **学生端**：iPad设备（Safari PWA / 原生APP），通过USB摄像头接入，核心完成语音对话、作业辅导、**题目指认**、AI交互（无端侧手指检测）
- **家长端**：**第一阶段以 Android 手机为主**（Flutter 原生 APK，小屏触控与推送更友好），完成远程查看、权限管控、远程介入教学、错题管理、试题集生成；**家长端 Web** 作为同一套业务后续补充（大屏查看、浏览器即用）

### 1.5 核心用户角色

- **学生**：K12阶段在校生，通过语音与AI对话，完成课业练习、试题批改、错题复习
- **家长**：学生监护人，远程管控学习过程、介入教学、审批错题、生成练习试题集

## 二、核心整体需求

1. 学生端登录后保持登录状态**一天以上**，无需频繁重新登录
2. 登录后直接进入**语音长聊天模式**，学生与AI通过实时语音进行自然对话，无发送按钮
3. AI根据对话内容**自动判断场景切换**（如学生说"我要开始做作业"，AI自动切换到摄像头模式）
4. 摄像头模式下支持左右分屏：左侧实时画面，右侧OCR识别结果
5. 学生可通过**语音 + 题目选中（点击/框选/说题号等，见 4.3.3）**与AI交互；**不在端侧运行手指关键点检测模型**
6. AI具备主动交互能力：遮挡提醒、对焦引导、确认是否需要帮助
7. 所有OCR结果按学生年级/科目/章节自动归类存储云端
8. 错题首先推送家长端，由家长决定是否同步学生端
9. 家长可远程介入教学，支持画中画视频交互
10. AI可命名、可选声音和讲话节奏，支持个性化配置

## 三、初始化与账号模块

### 3.1 初始化配置

- 首次使用需填写**学生年级**和**就读区域**（省/市/区三级下拉选项，数据内置于后端，通过 `/api/v1/regions/provinces`、`/cities`、`/districts` 接口获取）
- 大模型根据学生年级和区域信息，自动搜索对应的**学科目录信息**（如：三年级-语文-第一单元-课文1）
- 在云端建立该学生对应的**科目分类表**，后续所有OCR识别结果自动归入对应科目
- 初始化时引导学生或家长为AI命名（支持任意命名）
  - 限制规则：不允许使用亲属称谓（爸爸、妈妈、爷爷、奶奶、外公、外婆等通用家庭称呼）
  - AI以命名后的名字自称，增强亲切感
- 选择AI的**交互声音**和**讲话节奏**（如：温柔女声/活泼男声，语速快/中/慢）
- 以上配置在使用过程中可随时修改（设置入口常驻）

### 3.2 账号登录功能

- 双端支持**账号密码登录**，一套账号体系
- 登录后保持登录状态**至少24小时**，超过有效期后静默刷新 token，尽量避免要求重新登录
- 同一账号支持一台学生端 + 一台家长端同时在线
- 登录时自动识别设备角色（学生端/家长端）

### 3.3 数据同步

- 双端实时同步核心数据：学习记录、AI交互记录、OCR识别结果、错题集、权限设置
- 所有数据通过服务器中转，依托WebSocket实现实时推送
- 离线暂存，重新联网后自动同步

## 四、学生端功能性需求

### 4.1 语音长聊天模式（登录后默认界面）

- 登录后**直接进入语音聊天界面**，无需额外操作
- 交互方式为**实时语音对话**，类似面对面聊天：
  - 无发送按钮，语音实时采集
  - 根据学生**说话停顿时长**智能判断是否结束发言（如停顿1.5秒以上视为说完，AI开始回答）
  - 界面上实时显示**语音转文字**内容（学生和AI双方的对话文字均显示）
  - AI回答时通过语音播放，同时显示文字
- **聆听休眠与唤醒（省流与体验平衡）**  
  - 当学生**连续一段时间不说话**（时长可配置，如 30～120 秒）时，进入**聆听休眠**：停止向云端 STT 推流（或停止持续识别），界面可轻提示「已休眠，唤醒后可以继续说话」。  
  - 休眠后需通过**唤醒再说**恢复：支持**唤醒词**（如「小智小智」或与 AI 名字绑定的短语）或**按钮/轻触唤醒**；唤醒后重新进入正常聆听与对话。  
  - 休眠期间 **TTS 不播放**（除可选的简短休眠提示音，可用本地/预置音频，避免走云 TTS）。
- **语音识别（STT）接入节奏**  
  - **第一阶段（联调与测试）**：学生端优先接入 **Web Speech API（浏览器 SpeechRecognition）** 或系统提供的 **Speech** 相关能力，快速验证语音链路、对话流程与 UI。  
  - **正式阶段**：对接 **火山引擎** 等云端流式 STT，满足准确率、方言与持续优化需求；正式接入后仍须保留 **静音不传、VAD、休眠唤醒** 等策略（见第十三章）。
- **播报音色**  
  - 学生须在初始化或设置中为 AI **选择播报音色**（多种音色可选）；测试阶段若使用浏览器 **SpeechSynthesis** 播报，须**枚举当前环境可用发音人**供学生选择；正式阶段 **腾讯云 TTS** 的音色与前端选项须一一映射，保证「所选即所播」。
- 对话界面为标准聊天气泡样式，左侧为AI消息，右侧为学生消息
- 语音识别支持中文普通话，适配K12学生的口语表达习惯
- 所有对话记录上传云端存储

### 4.2 智能场景切换

- AI具备**场景感知能力**，通过大模型判断学生意图自动切换界面模式：
  - 学生说类似"我要开始做作业"、"帮我看看这张卷子"、"打开摄像头" → AI切换到**摄像头模式**
  - 学生说"关掉摄像头"、"我做完了"、"我想聊天" → AI切换回**聊天模式**
- 切换时AI语音确认："好的，我帮你打开摄像头，把试卷放在镜头前吧"
- 模式切换平滑过渡，语音对话在任何模式下都保持活跃

### 4.3 摄像头模式

#### 4.3.1 画面采集与对焦引导

- 进入摄像头模式后，浏览器实时显示USB摄像头采集的画面
- 支持选择摄像头（内置/USB外接）
- 画面上显示**对焦引导框**，提醒学生"请将书本或试卷对齐到框内"
- AI通过语音引导对焦："把试卷往左移一点"、"好的，已经对齐了"
- 对焦完成后自动触发OCR识别

#### 4.3.2 左右分屏模式

- 对焦完成后，画面切换为**左右分屏**：
  - **左侧**：USB摄像头实时画面（持续更新）
  - **右侧**：OCR识别结果（结构化显示识别出的题目）
- 当OCR识别到内容后，**右侧成为主画面**（占比更大）
- 语音对话切到后台保持实时活跃，学生随时可以语音提问
- 分屏比例可通过拖拽调整

#### 4.3.3 题目指认交互（无端侧手指检测）

> 不运行 MediaPipe 等端侧手部模型；指认语义由**交互 + 云端**完成。

- 学生以**语音**说明意图，同时用下列方式之一**选定题目**（可组合，详细设计选定默认路径）：
  1. **点击/点选**：在右侧 OCR 题目列表或左侧预览图上的**题目热区**点击（热区由最近一次整页/区域 OCR 返回的版面框生成）；
  2. **框选**：学生在预览画面上**拖拽矩形框**选定区域，上传该裁剪图或坐标，由**服务端**结合 OCR 版面解析对应题块；
  3. **语音题号**：学生说「第三题」「填空题第一题」等，由 **STT + 大模型** 与当前页 OCR 结构对齐（歧义时 AI 反问确认）。
- 示例："这道题怎么做？" → 须已选中一题或补一步点选/框选/题号 → AI 针对该题作答。
- AI 先确认意图再行动（与原流程一致）：
  - 学生："这道题怎么做？"
  - AI："你想让我教你解这道题吗？"
  - 学生："是的" → AI 开始讲解
- **可选增强（云端）**：若产品需要「指着卷面」的体验，可在学生**确认拍照后**将**静态帧或短视频片段**上传服务端，由 **DeepSeek Vision / 服务端视觉**估计指尖或指示区域（计费按 Vision 策略，见第十三章）；**不在 iPad 本地常驻手指跟踪模型**。

#### 4.3.4 遮挡检测与提醒（云端抽样判定）

- **不在端侧运行专用遮挡检测小模型**；在摄像头模式下，按**可配置间隔**上传**低帧率抽样帧**（或对焦完成后的对比帧），由 **服务端 / DeepSeek Vision** 判断画面是否大面积遮挡、是否失焦。
- 当判定**遮挡或无法辨认题目区域**时，AI 主动语音提醒（文案与原需求一致）：
  - "你挡住我了，我看不到题目啦"
  - "可以移动一下摄像头，或者换个姿势"
- 抽样恢复正常后 AI 确认："好了，现在看得清楚了"
- 须控制抽样频率与分辨率，与第十三章 Vision 次数上限、套餐档位一致。

### 4.4 AI批改与讲解

- 批改和讲解通过**语音 + 题目选中（点选/框选/题号）**进行，尽量减少键盘输入
- 批改流程：选定题目 → AI确认 → 显示批改结果（对/错、正确答案、解析）
- 讲解流程：AI通过语音讲解解题思路，同时在右侧面板高亮对应步骤
- AI讲解内容贴合学生认知水平，语言通俗易懂
- 错因分析自动分类：计算失误、审题不清、概念模糊、公式记错、步骤缺失等

### 4.5 AI个性化配置

- AI有名字，在对话中以自己的名字自称
- 学生或家长可随时修改：
  - AI名称
  - **AI播报音色**（多种音色可选；测试期对应 Web Speech API 可用发音人列表，正式期对应腾讯云 TTS 音色 ID，见 4.1）
  - AI讲话节奏（语速快/中/慢）
- 配置入口：聊天界面顶部设置图标，或通过语音命令修改

### 4.6 错题处理

- AI批改后的所有错题，自动收录并上传云端
- 错题**首先推送给家长端**，由家长决定是否同步到学生端
- 如果家长勾选"后续都推送"，则后续错题自动同时推送双端
- 错题包含：原题截图、学生答案、正确答案、错因分析、知识点标签、所属科目/章节

## 五、家长端功能性需求

### 5.1 远程介入教学（核心功能）

- 在学生做作业过程中，家长有权**远程接入**学生端
- 远程接入后的画面交互：
  - 家长画面以**画中画**形式显示在USB摄像头画面的**右上角**
  - 点击家长画面 → 家长画面**放大**顶替USB摄像头主画面，USB画面缩小至**左上角**
  - 再点击USB小画面 → USB画面恢复放大，家长画面缩回**右上角**
  - 两个画面可反复切换
- 家长远程介入后可以：
  - 与AI交流（AI同时理解家长和学生的语音）
  - 与学生交流
  - 将某道题推送给学生端
  - 将生成的试题集推送给学生端
- **家长介入后AI行为变化**：
  - AI自动减少主动发言
  - 除非AI被直接提问（家长或学生问AI），AI才回答
  - AI在家长介入时更多充当辅助角色

### 5.2 OCR识别结果管理

- 所有OCR识别结果**自动上传云端**
- 家长可以：
  - 按**日期**查看任意一天的识别试题
  - 按**科目**筛选查看
  - 查看该日的**错误题目**列表
  - 通过**大模型搜索**试题或错误题目（自然语言搜索，如"上周数学做错的分数题"）

### 5.3 题目自动归类

- 初始化时根据学生年级和就读区域，大模型自动建立**科目分类表**
- OCR识别的所有试题自动归入对应科目和章节
- 归类由大模型智能判断（根据题目内容识别所属科目、知识点、章节）
- 家长可查看完整的科目树结构和每个章节下的题目

### 5.4 错题集创建与试题生成

- 家长可以自主创建错题集：
  - 选择**科目**
  - 选择**时间段**（如：最近一周、本月）
  - 选择科目中的**具体章节**
- AI根据学生做错的试题，通过大模型**自动生成练习试题**
- 试题生成规则：
  - 家长可设置**正确题目与错误题目的比例**（即简单题vs针对错误知识点的题）
  - 默认比例：**正确2 : 错误8**（20%基础巩固题 + 80%针对薄弱点的题）
  - 也可选择"自动生成"，由AI根据学生情况自行决定比例
- **AI自审核机制**：
  - 生成的试题先经过大模型自身审核
  - 审核内容：题目是否符合年级水平、答案是否正确、是否与原错题知识点匹配
  - 审核不通过的题目自动重新生成替换
- 生成的试题集家长可以：
  - 推送给学生端
  - 选择打印

### 5.5 错题推送机制

- AI批改后的错题**首先推送家长端**
- 家长可以：
  - 逐条审核，选择是否推送到学生端
  - 勾选**"后续都推送"**：开启后，后续所有错题自动同时推送双端
- 推送到学生端的错题显示在学生端的错题列表中

### 5.6 权限管控

- AI解答权限：控制学生端是否可使用AI解答、讲解功能
- 权限修改实时同步学生端，立即生效
- 权限修改日志云端存储，可追溯

### 5.7 学习数据查看

- 查看学生完整学习记录：每日学习时长、题目完成量、正确率变化
- 查看所有科目的错题分布情况
- 查看AI交互记录

### 5.8 家长端 Android 手机形态（第一阶段）

- **载体**：主流 Android 手机（Flutter 客户端），官网提供 **APK 下载 / 二维码安装**；不要求家长先用电脑浏览器完成核心管控流程。
- **布局**：列表 + 详情为主，适配竖屏单手操作；远程介入、视频画中画在**小屏可切换全屏**，避免控件过小不可点。
- **推送（建议）**：错题待审、权限变更、学生求助等关键事件可接系统通知（具体实现列入详细设计）。
- **与学生端协同**：仍遵循「同一账号、学生端 iPad + 家长端一台手机同时在线」规则；后续 Web 端登录策略与 Android 一致。

## 六、AI核心服务功能性需求

### 6.1 语音对话服务

- 实时语音识别（STT）：将学生语音转为文字
- 语音合成（TTS）：将AI回复转为语音播放，支持多种音色和语速
- 对话上下文管理：维持长对话记忆，理解上下文
- 意图识别：判断学生是在闲聊、提问、请求批改、请求切换模式等
- 停顿检测：根据说话停顿时长判断是否发言结束

### 6.2 试题处理服务

- OCR识别：通过DeepSeek Vision识别印刷体和手写体，覆盖K12主流题型
- 自动批改：对照正确答案批改学生作答
- 题目讲解：生成文字+语音讲解，包含解题思路、知识点、易错点
- 错因分析：自动分类错误原因
- 知识点归类：识别题目所属科目、章节、知识点

### 6.3 题目指认与对齐服务（无端侧手指模型）

- 接收学生**点选/框选坐标**或**题号语义**，与 OCR 版面结构、题目 ID 对齐；必要时由大模型消歧。
- **可选**：对上传的静态帧由 **Vision** 估计指示区域，再映射到题目（走云端，非端侧小模型）。

### 6.4 遮挡检测服务（云端）

- 对抽样图像进行质量与遮挡判断（服务端或 Vision）
- 触发语音提醒与恢复确认；**不依赖端侧实时 CV 小模型**

### 6.5 试题生成与自审核服务

- 根据学生错题和薄弱知识点生成针对性练习题
- 支持按比例混合基础题和薄弱点题目
- 内置审核机制：验证题目质量、答案正确性、难度匹配度
- 审核不通过自动重新生成

### 6.6 智能搜索服务

- 支持自然语言搜索试题和错题（如"上周的数学错题"、"分数相关的题目"）
- 搜索范围覆盖所有云端存储的OCR识别结果

## 七、双端协同核心流程

### 7.1 日常学习流程

1. 学生登录 → 进入语音聊天界面 → 与AI自由对话
2. 学生说"我要做作业" → AI切换到摄像头模式
3. 学生放置试卷 → AI引导对焦 → OCR识别 → 进入左右分屏

5. 家长可推送题目或试题集给学生

### 7.3 错题集生成流程

1. 家长选择科目 + 时间段/章节
2. AI根据错题生成练习试题（按设定比例）

4. 家长预览 → 推送学生端 / 打印

## 十三、API 成本控制与上下文记忆策略

> 本节为**产品与算法侧约束**，用于压低 STT / DeepSeek / Vision / TTS 的调用量与 token 消耗，与第十二章「后续单独补充的非功能需求」互补，实施时以本节为准。

### 13.1 语音识别（STT）——静音不传、按说话计费

- **静音不传**：检测到静音或低于阈值的音频帧不向 STT 服务发送（或发送间隔拉长至可配置上限）。
- **VAD 截断**：采用**非神经网络**的语音活动策略亦可接受（如短时能量/过零率阈值 + 停顿计时），或由**云端 STT 服务返回的分段结果**截断；**不在端侧部署神经网络 VAD 小模型**。目标不变：仅在有效说话片段内计费。
- **只在说话时识别**：未检测到说话时，不维持高频率的连续流式识别；可与「长按说话 / 唤醒后聆听」等交互策略组合（具体交互形态可在详细设计中细化，但**计费语义**须满足：无有效语音则不计或少计 STT 时长）。
- **接入路径（与第四章一致）**  
  - **测试阶段**：使用 **Web Speech API（SpeechRecognition）** 或系统 Speech 能力完成端到端验证（无火山密钥亦可联调）。  
  - **正式阶段**：切换为 **火山引擎流式 STT**；策略（静音、VAD、休眠）双阶段均需遵守。  
- **与「音色」的关系**：STT 负责**转文字**；**音色**指 **TTS 播报声线**。测试阶段通过 **SpeechSynthesis** 选择本地可用发音人；正式阶段通过 **腾讯云 TTS** 选音色，二者均须在设置页提供可选列表（见 13.4）。

### 13.2 大模型上下文（DeepSeek）——当日全量、跨日压缩记忆

- **当日上下文**：学生当天与 AI 的对话上下文在服务端**完整保存**（用于连贯辅导与复盘），可按会话/自然日切分。
- **日终压缩**：每个自然日结束后，由服务端调度（可批任务）对当日对话做**大义提取与摘要**，生成一条（或多条结构化）**长期记忆**，体积目标为当日原文信息量的 **约 1%～0.1%**（即压缩至百分之一至千分之一量级，具体比例可配置，但须显著小于原文 token）。
- **记忆存储**：压缩后的记忆持久化在服务器（数据库或对象存储+索引，由详细设计定），与用户账号绑定。
- **对话加载规则**：学生**当天**发起对话时，除当日已累计的上下文外，须自动加载**最近 3 个自然日**已生成的压缩记忆（不含当日未完成压缩的部分）；超过 3 日的历史仅通过压缩记忆间接参与，避免上下文无限膨胀。

### 13.3 视觉 / OCR（DeepSeek Vision 等）——按需两次识别

- **不在视频流上持续、高频调用多模态 OCR**，以控制图片 token 与调用次数。
- **第一次：选定题目时识别**  
  学生通过**点选/框选/说题号**完成题目定位并发起提问时，对该次交互所需的画面区域/截图执行题目区域识别与结构化解析（具体是整页裁剪还是局部截图由详细设计定，但须满足「选定题目动作触发」）；**可选**上传静态帧由云端 Vision 辅助指示区域（仍无端侧小模型）。
- **第二次：批改时识别**  
  进入批改流程时，对用于判分的画面/截图再次识别（可与指认结果合并校验，避免重复全页识别；若批改与指认共用同一次截图且内容未变，**应复用识别结果**，不重复调用 Vision）。

### 13.4 语音合成（TTS）——只朗读「提炼后的简洁答复」

- **朗读对象**：TTS 播放的内容**不是** DeepSeek 返回的完整长文默认稿，而是经服务端或模型二次处理后的**简洁版答复**（提炼要点、步骤条数控制、字数上限可配置）。
- **流程要求**：建议链路为「DeepSeek 生成完整回答 → 生成/抽取 `tts_text`（短）→ 腾讯云 TTS 仅对 `tts_text` 计费」。界面可仍展示完整版（图文），与朗读内容可以不一致。
- **目的**：在保持辅导质量的前提下，显著降低 TTS **字符数**与听感上的冗长。
- **休眠与唤醒（与 4.1 一致，侧重计费与实现）**  
  - 学生**持续无有效语音**超过阈值后，进入**聆听休眠**：**停止**向云端推送需计费的 STT 音频流（或结束识别会话）；TTS 除可选本地提示外**不再调用云 TTS**。  
  - **唤醒再说**：通过唤醒词或显式操作恢复聆听后，再重新建立 STT 会话；从休眠到唤醒之间**不产生**流式 STT 时长费用。  
- **音色选择**  
  - **测试阶段**：使用 **SpeechSynthesis** 时，须拉取并展示 `speechSynthesis.getVoices()`（或系统等价能力）中的中文等可用发音人，供与「AI 声音」配置联动。  
  - **正式阶段**：使用 **腾讯云 TTS** 时，须在后台维护**音色列表**（音色 ID、展示名、试听样例可选），与前端选项一致；学生变更音色后立即作用于后续 `tts_text` 播报。

### 13.5 验收与配置项（建议）

- STT：可配置静音阈值、VAD 起止灵敏度、最大单次聆听时长；**休眠无语音时长阈值**、**唤醒方式**（唤醒词列表 / 仅按钮）。
- 记忆：可配置日终压缩任务时间、压缩比目标区间（1%～0.1%）、加载历史记忆天数（默认 3）。
- Vision：记录「指认触发次数」「批改触发次数」及复用命中率，用于成本审计。
- TTS：`tts_text` 最大字数/最大条数、是否与展示全文脱钩；**音色列表**与映射表版本管理。

## 十四、会员分级套餐（控制云资源与商业化）

> 与第十三章技术策略叠加：套餐限制「云 STT 时长、Vision 次数、视频介入」等硬上限；策略负责「同样额度内尽量少调用」。具体数值由运营在后台或配置中心调整，下表为**示例默认值**，上线前可再校准。

### 14.1 设计原则

- **可感知**：超限时 UI 明确提示原因与恢复时间（次日重置 / 升级套餐），避免无声失败。
- **可降级**：超限后优先 **端侧 Speech API** 仅出字、或 **仅文字交互**、或 **异步批改排队**，而非直接阻断学习。
- **家长可见**：家长端可查看当前套餐余量（可选）及升级入口（若商业化）。

### 14.2 套餐档位（示例）

| 项目 | 体验版 | 学习版 | 畅学版 |
|------|--------|--------|--------|
| **定位** | 拉新试用 | 日常主力 | 高频/多科 |
| **每日云 STT 等效时长**（分钟） | ≤ 15 | ≤ 60 | ≤ 180 |
| **每日 Vision 调用次数**（指认+批改合计，可复用不计次规则在详细设计中定义） | ≤ 10 | ≤ 40 | ≤ 120 |
| **每日 DeepSeek 对话轮次上限**（可选，0 表示不单独限轮次而靠前两项约束） | ≤ 30 | ≤ 120 | ≤ 400 |
| **云 TTS 字符/日**（可选上限，`tts_text` 计数字符） | ≤ 5 000 | ≤ 2 万 | ≤ 8 万 |
| **家长远程视频介入** | 关闭或 ≤ 5 分钟/日 | ≤ 30 分钟/日 | ≤ 120 分钟/日 |
| **错题推送 / 大模型搜题等增值能力** | 基础 | 标准 | 全开 |
| **异步批改（非实时队列）** | 仅低价任务 | 支持 | 优先队列 |

说明：「每日」可按自然日 0 点重置，或与用户首次登录滚动 24 小时，由详细设计选定一种并写清时区。

### 14.3 超限与降级策略（示例）

- **云 STT 用尽**：自动提示；后续改用 **Web Speech API 仅本地识别**（若环境支持）或 **键盘输入**；次日或升级后恢复云端识别。
- **Vision 用尽**：允许继续**文字与语音问答**，拍照/指认入口灰显并说明原因；已缓存的 OCR 结果仍可查看。
- **TTS 用尽**：AI 回答**仅展示文字**；可选 **SpeechSynthesis 本地朗读**（受设备音色限制）。
- **视频介入用尽**：家长端入口提示；学生端画中画关闭或仅音频（若允许）。

### 14.4 与账号的关系

- 套餐与用户账号或「家庭订阅」绑定；学生端、家长端展示一致档位。
- 管理后台（或未来计费系统）可配置各档位阈值，无需发版即可调整。

### 14.5 验收要点

- 各档位在达到阈值时，**拦截点**与**提示文案**符合上表逻辑。
- 统计口径：STT 按**有效推流时长**、Vision 按**成功发起且未命中复用缓存**的次数（若采用复用规则）、TTS 按**实际合成字符数**。
