# 学习指认AI — 集成测试文档

**产品名称**：学习指认AI（K12 AI 辅助学习平台）  
**文档版本**：1.1  
**适用范围**：端到端 API、双端协同、部署与环境验证  

---

## 1. 文档目的

本说明定义「学习指认AI」在**接近真实环境**下的集成与验收测试策略，覆盖 HTTP API 链路、学生/家长双端协同、管理配置与上线检查项，与《单元测试文档》形成互补：单元测试保证模块正确性，集成测试保证**系统串联与部署可用**。

---

## 2. 测试策略

### 2.1 总体方法

| 维度 | 方法 | 工具 / 说明 |
|------|------|-------------|
| 端到端 API 测试 | 对真实或类生产环境发起完整请求链，使用持久化存储与真实（或沙箱）AI 配置 | **httpx**（异步）或 **requests**；pytest 组织场景化用例 |
| 双端协同场景测试 | 同一账号学生端 + 家长端（或模拟双客户端）验证权限、推送、状态一致 | 可结合 WebSocket 客户端脚本与 REST 交替调用 |
| 部署验证测试 | 发布后对端口、健康检查、文档、核心接口做冒烟 | 自动化脚本（如 `scripts/verify_deployment.py`）+ 人工抽检 |

### 2.2 环境要求

- 测试环境与生产隔离；数据库迁移已执行；Redis、对象存储（若启用）可达。
- 敏感配置（API Key）使用测试密钥，避免污染生产数据。
- 建议固定测试账号与可清理的会话数据，避免用例间相互污染（或用例内自注册并 teardown）。

### 2.3 通过标准（建议）

- 本文档所列**端到端场景**在目标环境全部通过。
- **部署验证清单**中关键项（健康检查、注册登录、AI 对话、数据库）无失败项。
- 阻塞级缺陷（P0）为零后方可发布。

## 3. 端到端测试场景

### 3.1 场景 1：学生完整学习流程

| 步骤 | 操作 | 验证点 |
|------|------|--------|
| 1 | 注册新账号 | 返回成功，可用新账号登录 |
| 2 | 登录（学生端） | 获得有效 Token，`GET /api/v1/auth/me` 信息正确 |
| 3 | 初始化设置（年级、区域、AI 名字） | `POST /api/v1/init/setup` 成功；禁用亲属称谓时接口拒绝非法昵称 |
| 4 | 发送对话消息，验证 AI 回复 | `POST /api/v1/chat/message` 返回非空回复；延迟在可接受范围 |
| 5 | 发送「我要做作业」，验证场景切换 | 响应或后续上下文中模式为 camera（或与产品一致的字段） |
| 6 | 上传试卷图片，验证 OCR 识别 | `POST /api/v1/questions/ocr`（或先 `upload-image` 再 OCR）返回可读题干 |
| 7 | 批改试题，验证结果 | 创建题目后调用批改接口，结果字段完整、状态正确 |
| 8 | 验证错题推送 | 存在错题时，错题集生成或通知链路符合设计（家长端可见待审批或消息到达） |

**成功准则**：从注册到错题相关能力全链路无 5xx，业务状态与数据库一致。

---

### 3.2 场景 2：家长权限管控流程

| 步骤 | 操作 | 验证点 |
|------|------|--------|
| 1 | 登录（家长端） | 家长角色 Token 有效（若与学生分端账号，则分别登录） |
| 2 | 查看权限设置 | `GET /api/v1/permissions/` 返回当前开关 |
| 3 | 关闭 AI 解答权限 | `PUT /api/v1/permissions/` 更新成功 |
| 4 | 学生端验证权限生效 | 学生端请求解答类接口返回 403 或业务禁止码（与产品一致） |
| 5 | 查看学习记录 | `GET /api/v1/study-records/` 等返回预期列表或空列表结构正确 |

**成功准则**：权限变更持久化，且学生侧行为与配置实时或准实时一致（参见双端协同测试）。

---

### 3.3 场景 3：错题集审批流程

| 步骤 | 操作 | 验证点 |
|------|------|--------|
| 1 | AI 生成错题集 | `POST /api/v1/error-books/generate?session_id=...` 成功，状态为待审批 |
| 2 | 错题推送家长端 | WebSocket 或轮询侧收到 `error_book_pending` 类事件（与实现一致） |
| 3 | 家长审批通过 / 驳回 | `POST /api/v1/error-books/{id}/approve`，`approved` true/false 分支均验证 |
| 4 | 驳回后自动重新生成 | 驳回后调用 `POST .../regenerate`，状态回到待审批，内容有更新预期 |
| 5 | 审批通过后学生端可见 | 学生端 `GET /api/v1/error-books/{id}` 或列表可见已通过记录 |

**成功准则**：状态机：`PENDING_APPROVAL` → `APPROVED` / `REJECTED` →（驳回）`REGENERATING` → `PENDING_APPROVAL` 符合接口约束。

---

### 3.4 场景 4：管理后台

| 步骤 | 操作 | 验证点 |
|------|------|--------|
| 1 | 管理员登录 | 返回管理员凭证 |
| 2 | 查看 API Key 配置 | 响应为脱敏展示 |
| 3 | 修改 API Key | 更新成功 |
| 4 | 验证配置生效 | 后续 Chat/Vision 调用使用新密钥成功（或沙箱验证） |
| 5 | 查看平台统计数据 | 指标字段完整、数值合理 |

> **说明**：管理后台 API 若以独立服务或路由挂载，请将下表中的 `/api/v1/admin/*` 替换为实际前缀；未实现前本场景可作为验收清单保留，并在 CI 中标记为 `xfail` 或跳过。

**成功准则**：配置变更可观测、可回滚，统计接口无越权访问。

## 4. 双端协同测试

| 测试项 | 方法 | 预期 |
|--------|------|------|
| 同一账号双端同时在线 | 两个客户端分别建立会话（HTTP + WS） | 均无异常踢线；会话隔离符合设计 |
| WebSocket 消息同步 | 一端触发事件，另一端订阅 `/ws` | 收到对应业务类型 payload |
| 权限实时推送 | 家长修改权限 | 学生端在约定时间内收到更新（WS 或下次请求生效） |
| 错题集审批状态同步 | 家长审批后 | 学生端列表/详情状态与数据库一致 |

**建议**：为 WebSocket 编写独立 `pytest` 模块或使用 `websockets` 客户端脚本，断言 JSON 结构与 `error_book_*`、`permission_*` 等事件名。

## 6. 部署验证检查清单

在目标服务器或容器环境完成发布后，按下列项勾选验证（可根据 nginx 实际端口调整）。

- [ ] 官网（**80** 端口）可访问
- [ ] PWA（**8000** 端口，或与 nginx 反代一致）可访问
- [ ] API 健康检查通过（`GET /health`）
- [ ] Swagger 文档可访问（`GET /docs`）
- [ ] WebSocket 连接正常（`WS /ws`）
- [ ] 用户注册 / 登录正常
- [ ] AI 对话接口正常
- [ ] OCR 识别接口正常
- [ ] 数据库表创建完整（迁移无缺失）
- [ ] Redis 连接正常
- [ ] 管理后台登录正常
- [ ] API Key 管理功能正常
- [ ] **systemd** 服务异常退出后能自动重启（`Restart=on-failure` 等配置）

## 7. 执行方式

### 7.1 远程 HTTP 冒烟（pytest 标记 `integration`）

仓库实现：`server/tests/test_integration_remote.py`。

```bash
# 指向已部署服务（示例：经 Nginx 访问 FastAPI）
set INTEGRATION_BASE_URL=http://101.201.244.150:8000
set INTEGRATION_ADMIN_PASSWORD=你的超管密码
cd server && pytest tests/test_integration_remote.py -v -m integration
```

- **`INTEGRATION_BASE_URL`**：必填（不含末尾 `/`）。可用 **8000**（应用入口）或内网直连 **8001**（仅服务器本机）。  
- **`INTEGRATION_ADMIN_USER`**：可选，默认 `admin`。  
- **`INTEGRATION_ADMIN_PASSWORD`**：与服务器 `.env` 中 `ADMIN_PASSWORD` 一致；未设置则跳过需要登录的用例。

### 7.2 部署脚本内置门禁

`scripts/deploy_all.py` 在上传代码后、重启服务前，于服务器上执行：

- `export DATABASE_URL=sqlite+aiosqlite:///:memory:` 等测试环境变量  
- `python -m pytest tests/ -m "not integration"`  

失败则**中止发布**（不重启 `ai-study`）。

### 7.3 其他脚本

```bash
python scripts/verify_deployment.py   # 从本机探测服务器端口（需网络可达）
```

## 8. 与单元测试的边界

| 集成测试 | 单元测试 |
|----------|----------|
| 多接口、真实中间件、真实或集成 DB | 单模块、Mock 外部系统 |
| 运行慢、环境依赖强 | 运行快、可本地频繁执行 |
| 发布前 / 定时全量 | 每次提交 |
