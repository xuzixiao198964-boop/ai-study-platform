# 学习指认AI产品 — 功能性需求文档

## 一、文档概述

### 1.1 项目背景

本项目针对K12阶段学生课业辅导、错题巩固的核心需求，打造学生端（iPad）+ 家长端（Android）双端协同的学习指认AI产品。依托iPad外接USB摄像头实现试题实时投屏，结合AI技术完成试题识别、批改、讲解、错题生成，**支持手指指认功能（学生用手指指向试题，AI自动识别指向的题目并处理）**。同时支持家长远程查看、权限管控、错题审批与协同指导。双端通过同一账号登录实现数据同步，错题集数据统一存储至服务器，打造家校/亲子协同的个性化学习辅导工具。

### 1.2 核心目标

- 实现iPad端USB摄像头试题实时投屏、**手指指认交互**、语音交互、AI试题处理全流程操作
- 搭建Android端家长远程管控体系，实现权限控制、实时查看、协同指导
- 双端登录同一账号自动完成数据同步，学生端错题集数据统一存储于服务器
- 完成AI错题集自动生成、双重校验、家长审批及重新生成的闭环流程
- 落地AI题目讲解、错题相似题推送、亲子+AI联合指导核心功能
- **实现手指指认识别能力：学生用手指指向摄像头画面中的题目，AI自动定位并选中该题**

### 1.3 适用范围

本文档适用于产品研发、设计、开发、测试全流程人员，明确产品功能性需求，暂不包含性能、兼容性、安全等非功能性专项要求。

### 1.4 双端定位

- **学生端**：iPad设备，通过USB-C/Lightning转接器接入USB摄像头，核心完成试题投屏、**手指指认选题**、答题、AI交互、接收家长指导，登录账号后同步家长端数据
- **家长端**：Android手机/平板设备，核心完成远程实时查看、权限管控、错题审批、远程指导，登录账号后同步学生端数据

### 1.5 核心用户角色

- **学生**：K12阶段在校生，使用iPad端完成课业练习、试题批改、错题复习
- **家长**：学生监护人，使用Android端远程管控学习过程、审批错题内容、协同辅导

## 二、核心整体需求

1. 学生端iPad支持USB摄像头即插即用，实时投屏试题画面，支持**手指指认+语音+点击**三种交互方式
2. **手指指认功能**：摄像头实时检测学生手指位置，当手指指向某道题目时，AI自动识别并选中该题，触发后续处理流程
3. AI具备试题识别、自动批改、题目讲解、错因分析、错题集生成、相似题推送能力
4. 家长端与学生端登录同一账号，自动实现设备绑定与数据同步
5. 家长端Android可远程实时查看学生端试题投屏画面，全程管控学习操作权限
6. 错题集由AI自动生成后，需提交家长端审批，家长可通过或驳回并要求重新生成
7. 学生端所有错题集数据统一存储至服务器，双端可实时调取查看
8. 支持家长+AI联合指导模式，家长可自定义AI解答权限，实现个性化学习管控
9. **支持家长与学生之间的实时视频通话**，家长可远程面对面指导学生学习

## 三、账号登录与数据同步模块

### 3.1 账号登录功能

- 双端支持**账号密码登录**方式，仅需一套账号体系，无独立家长/学生账号区分
- 家长端与学生端登录同一个账号，即默认完成设备绑定与数据关联，无需额外扫码/手动绑定操作
- 支持账号退出登录、密码修改、账号注销基础操作，退出后自动断开数据同步
- 同一账号仅支持一台学生端（iPad）+ 一台家长端（Android）同时在线登录

### 3.2 数据同步功能

- 登录同一账号后，双端实时同步核心数据：学生学习记录、试题投屏画面状态、AI批改结果、错题集生成与审批状态
- 学生端操作数据（答题、错题生成、AI交互记录）实时上传服务器，同步推送至家长端
- 家长端操作数据（权限设置、错题审批、远程指令）实时上传服务器，同步推送至学生端
- 离线状态下，双端本地暂存操作数据，重新联网登录账号后，自动同步至服务器并更新双端数据

### 3.3 服务器数据存储

- 学生端所有错题集数据100%存储于服务器，包含：原题截图、错误答案、正确答案、错因分析、知识点标签、审批状态、生成时间
- 服务器同步存储账号关联的所有数据：学习记录、AI交互记录、权限设置记录、错题审批历史
- 双端登录账号后，可直接从服务器调取对应数据，无需本地缓存完整数据，更换设备登录同一账号，可加载全部历史数据

## 四、学生端（iPad）功能性需求

### 4.1 硬件接入与试题投屏模块

- 支持USB摄像头通过iPad转接器即插即用，自动识别摄像头设备，无需额外配置
- 摄像头采集试题画面后，实时全屏/半屏投屏至iPad端，支持画面缩放、对焦、裁剪（仅保留试题区域）
- 投屏画面自动框选单道试题区域，标注试题序号，支持点击单题选中操作
- 投屏过程中可随时切换摄像头开关、画面显示模式，操作便捷无复杂步骤
- 投屏画面及操作状态，实时通过账号同步至服务器，推送至家长端

### 4.2 手指指认模块（新增核心功能）

- **手指检测**：利用MediaPipe手势检测模型，实时检测摄像头画面中学生手指的位置和指向
- **指向识别**：当学生用食指指向投屏画面中的某道题目时，系统自动计算指尖坐标与题目区域的映射关系，高亮选中对应题目
- **指认交互流程**：
  1. 摄像头捕获画面 → 同时进行试题区域检测和手指检测
  2. 手指指向某题区域并停留1秒 → 系统确认选中该题 → 语音/视觉反馈"已选中第X题"
  3. 选中后可继续通过语音或点击触发AI批改、讲解等操作
- **指认状态显示**：画面上实时显示手指追踪光标，选中题目时高亮边框
- 指认操作记录上传服务器，同步至家长端

### 4.3 账号登录与数据同步模块

- 登录界面与家长端共用同一账号入口，输入同一账号完成登录，自动关联家长端设备
- 登录后自动从服务器同步家长端权限设置、历史错题数据、学习记录
- 错题生成、作答、AI批改等数据，实时上传至服务器存储，同步家长端
- 退出账号后，清空本地临时数据，断开与家长端同步连接

### 4.4 语音交互模块

- 内置语音唤醒功能，支持自定义唤醒词，唤醒后可接收语音指令
- 支持基础语音指令：开始识别试题、暂停投屏、批改本题、AI讲解本题、生成错题集、查看相似题
- 语音指令执行后，实时语音播报执行结果，无响应时提示重新输入指令
- 语音交互全程伴随投屏画面同步，不影响试题显示与操作，交互记录上传服务器

### 4.5 试题AI处理模块

- **试题识别**：AI自动识别投屏试题，支持印刷体、手写体答案混合识别，覆盖选择、填空、计算、应用等K12主流题型，识别结果同步显示在试题旁，记录上传服务器
- **AI批改**：点击单题（或指认选中）后，AI自动批改学生答案，标注正确/错误，显示标准答案与解题步骤，支持学生手动修正AI批改结果，批改数据实时同步服务器
- **AI题目讲解**：AI针对错题/疑问题，生成文字+语音讲解，包含解题思路、知识点解析、易错点提醒、步骤拆解，可重复播放讲解内容，讲解记录上传服务器
- **AI解答权限联动**：学生端AI解答功能受家长端权限控制，权限关闭时，点击AI讲解/解答按钮，提示"家长已关闭AI解答权限，请联系家长"，权限状态从服务器同步获取

### 4.6 错题集相关模块

- AI自动收录批改后的错题，按科目、知识点、错因类型分类整理，生成初始错题集草稿
- 错题集包含：原题截图、学生错误答案、正确答案、错因分析、知识点标签，所有内容实时上传服务器存储
- AI生成错题集后，自动提交至服务器，推送至同一账号的家长端等待审批，学生端显示"错题集待家长审批"状态，不可直接使用
- 接收服务器同步的家长端审批结果：审批通过则错题集在服务器标记为生效，学生端可调取查看；审批驳回则显示"家长要求重新生成"，自动触发AI重新生成错题集，新草稿再次上传服务器提交审批
- 生效后的错题集从服务器调取查看、复习，关联推送相似练习题，可直接在线作答，作答记录同步服务器

### 4.7 远程协同模块

- 通过服务器中转，实时向同一账号的家长端Android同步试题投屏画面
- **支持与家长端的实时视频通话**：学生可发起或接受家长的视频通话请求，通过声网Agora SDK实现低延迟音视频传输
- 接收家长端远程操作指令（如：聚焦某道题、重新识别试题），自动执行对应操作
- 显示家长在线/离线状态，状态数据从服务器同步，家长在线时可随时发起协同指导

## 五、家长端（Android）功能性需求

### 5.1 账号登录与数据同步模块

- 登录界面与学生端共用同一账号入口，输入与学生端相同的账号，登录后自动完成数据同步
- 登录后自动从服务器调取学生端设备状态、学习记录、历史错题、权限设置数据
- 权限修改、错题审批等操作，实时上传服务器，同步推送至同一账号的学生端
- 支持查看服务器存储的所有历史学习数据、错题数据、审批记录，退出账号后断开同步

### 5.2 远程实时查看模块

- 通过服务器同步，实时查看学生端iPad试题投屏画面，与学生端显示内容完全一致
- 画面同步支持全屏查看，可查看学生操作记录、AI批改结果、试题识别内容
- **支持与学生端的实时视频通话功能**：家长可发起视频通话，面对面远程指导学生，通过声网Agora SDK实现，支持视频录制、截图

### 5.3 核心权限管控模块

- AI解答权限总控：设置独立开关，控制学生端是否可使用AI解答、AI题目讲解功能
  - 开启：权限状态上传服务器，同步至学生端，学生端可正常使用所有AI解答、讲解功能
  - 关闭：权限状态上传服务器，同步至学生端，学生端所有AI解答、讲解入口屏蔽，无法触发相关操作
- 权限修改实时通过服务器同步至学生端，修改后立即生效，无需重启设备
- 权限修改日志上传服务器存储，双端可追溯查看修改时间、修改内容

### 5.4 错题集审批模块

- 从服务器接收学生端提交的AI生成错题集，推送审批提醒，显示错题集完整内容（科目、错题数量、错因分析、知识点、原题截图）
- 审批操作：
  - **通过**：审批结果上传服务器，标记错题集为生效状态，同步至学生端，学生端可从服务器调取使用
  - **驳回并要求重新生成**：填写驳回原因（可选），审批结果上传服务器，同步至学生端，触发AI重新生成错题集，重新生成后再次上传服务器提交审批
- 支持从服务器调取查看历史错题集审批记录，包含审批时间、审批结果、错题集内容，可重新查看历史审批通过的错题集
- 家长可手动编辑审批通过的错题集，修改内容上传服务器更新，同步至学生端

### 5.5 协同指导模块

- 支持远程标注功能：家长在实时投屏画面上标注试题重点、解题思路，标注内容通过服务器同步显示至学生端iPad
- 可与AI联合指导：家长查看服务器同步的AI批改、讲解结果，结合自身指导意见，向学生发送个性化指导内容
- 从服务器调取查看学生完整学习记录：试题批改情况、错题集生成与审批记录、AI讲解使用记录

## 六、AI核心服务功能性需求

### 6.1 试题处理服务

- 支持K12全学科试题识别、批改，适配各年级知识点与题型
- 题目讲解内容贴合学生认知水平，语言通俗易懂，步骤清晰无超纲内容
- 错因分析自动分类：计算失误、审题不清、概念模糊、公式记错、步骤缺失、逻辑错误等
- 试题处理结果实时上传服务器，同步双端

### 6.2 错题集生成与校验服务

- 自动抓取学生错题，按规则整理生成结构化错题集，确保无漏题、无错录
- 内置AI校验机制：校验错题与学生作答内容匹配度、知识点标签准确性、错因分析合理性
- 生成的错题集实时上传服务器，推送家长端审批；接收家长端重新生成指令后，快速重新梳理错题，优化内容后再次上传服务器提交审批
- 所有错题集数据永久存储于服务器，支持双端随时调取、修改、更新

### 6.3 相似题生成服务

- 针对服务器中审批通过的错题，按同知识点、同难度、同题型生成相似练习题
- 数量可自定义，题目贴合教材考纲，无重复、无超纲题目
- 相似题数据存储于服务器，同步双端

### 6.4 手指指认识别服务

- 基于MediaPipe手势检测，实时识别摄像头画面中的手指位置和指向方向
- 将指尖坐标映射到试题区域，确定学生指向的具体题目
- 支持指认确认机制：手指在目标区域停留超过1秒视为确认选择
- 指认结果与OCR识别结果联动，自动触发选中题目的后续AI处理流程

## 七、双端协同核心流程说明

1. **登录同步流程**：家长端/学生端分别登录同一账号 → 自动连接服务器 → 完成设备绑定与数据同步 → 双端实时共享数据
2. **学习流程**：学生端登录账号 → 连接摄像头 → 投屏试题 → **手指指认选题/语音/点击操作** → AI识别批改（权限从服务器同步）→ AI生成错题集并上传服务器 → 服务器推送家长端审批 → 家长审批结果同步服务器 → 学生端从服务器调取生效错题集复习
3. **家长管控流程**：家长端登录同一账号 → 从服务器同步学生端数据 → 远程查看投屏 → 设置AI权限并上传服务器 → 审批服务器中的错题集 → **远程视频通话/标注指导** → 查看服务器存储的学习记录
4. **错题闭环流程**：AI生成错题 → AI内部校验 → 上传服务器 → 推送家长审批 → 家长通过（服务器标记生效）/ 驳回（服务器触发重新生成）→ AI重新生成 → 再次上传服务器审批

## 八、技术选型

| 层级 | 技术 | 说明 |
|---|---|---|
| 学生端 (iPad) | Flutter | 跨平台框架，支持iOS摄像头接入 |
| 家长端 (Android) | Flutter | 与学生端共享大部分业务代码 |
| 后端服务 | Python FastAPI | 高性能异步API框架 |
| 数据库 | PostgreSQL + Redis | 持久化存储 + 缓存/实时推送 |
| 实时通信 | WebSocket | 双端数据同步、实时推送 |
| 视频通话 | 声网Agora SDK | 低延迟实时音视频通话 |
| 文件存储 | MinIO | 本地对象存储，存储试题截图等 |
| AI全能力 | DeepSeek API (含Vision多模态) | OCR识别+试题解析+批改+讲解+错因分析+相似题生成，统一由DeepSeek提供 |
| 手指检测 | MediaPipe | 手势/手指位置实时检测 |
| 部署方式 | 裸机部署 | systemd管理服务进程 |

## 九、补充说明

- 本文档仅明确功能性需求，性能、并发、兼容性、安全、服务器运维等非功能性需求，后续单独补充专项文档
- 双端操作界面需简洁易懂，学生端侧重便捷操作，家长端侧重管控与查看，符合用户使用习惯
- 所有数据交互均通过服务器中转，依托同一账号实现同步，确保数据一致性
- 所有功能需围绕亲子协同、AI辅助学习核心，满足家长对孩子学习的管控与辅导需求
