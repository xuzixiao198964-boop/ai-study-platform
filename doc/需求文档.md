# 学习指认AI产品 — 功能性需求文档

## 一、文档概述

### 1.1 项目背景

本项目针对K12阶段学生课业辅导、错题巩固的核心需求，打造学生端（iPad）+ 家长端（Android/Web）双端协同的学习指认AI产品。学生端以**语音对话**为核心交互方式，结合USB外接摄像头实现试题实时投屏与指认，通过AI完成试题识别、批改、讲解、错题生成。家长端实现远程查看、权限管控、错题审批、远程介入教学与协同指导。双端通过同一账号登录实现数据同步，所有数据统一存储至云端。

### 1.2 核心目标

- 以**语音实时对话**为主要交互方式，学生与AI进行自然语言交流，无需手动输入
- AI具备情境感知能力，根据对话内容自动切换聊天模式与作业辅导模式
- 通过USB摄像头实现试题实时投屏、OCR识别、手指指认交互
- AI遮挡检测：当学生遮挡摄像头画面时，AI主动提醒
- 搭建家长端远程管控体系，支持远程介入教学、画中画交互、权限控制
- 所有OCR识别结果按学生年级、科目、章节自动归类存储至云端
- 家长端可按科目/时间/章节创建错题集，AI自动生成带审核机制的练习试题
- AI可命名、可选声音和讲话节奏，实现个性化交互体验

### 1.3 适用范围

本文档适用于产品研发、设计、开发、测试全流程人员，明确产品功能性需求。

### 1.4 双端定位

- **学生端**：iPad设备（Safari PWA / 原生APP），通过USB摄像头接入，核心完成语音对话、作业辅导、试题指认、AI交互
- **家长端**：Android/Web设备，核心完成远程查看、权限管控、远程介入教学、错题管理、试题集生成

### 1.5 核心用户角色

- **学生**：K12阶段在校生，通过语音与AI对话，完成课业练习、试题批改、错题复习
- **家长**：学生监护人，远程管控学习过程、介入教学、审批错题、生成练习试题集

## 二、核心整体需求

1. 学生端登录后保持登录状态**一天以上**，无需频繁重新登录
2. 登录后直接进入**语音长聊天模式**，学生与AI通过实时语音进行自然对话，无发送按钮
3. AI根据对话内容**自动判断场景切换**（如学生说"我要开始做作业"，AI自动切换到摄像头模式）
4. 摄像头模式下支持左右分屏：左侧实时画面，右侧OCR识别结果
5. 学生可通过**语音+手指指认**的方式与AI交互（如指着某题说"这道题怎么做"）
6. AI具备主动交互能力：遮挡提醒、对焦引导、确认是否需要帮助
7. 所有OCR结果按学生年级/科目/章节自动归类存储云端
8. 错题首先推送家长端，由家长决定是否同步学生端
9. 家长可远程介入教学，支持画中画视频交互
10. AI可命名、可选声音和讲话节奏，支持个性化配置

## 三、初始化与账号模块

### 3.1 初始化配置

- 首次使用需填写**学生年级**和**就读区域**
- 大模型根据学生年级和区域信息，自动搜索对应的**学科目录信息**（如：三年级-语文-第一单元-课文1）
- 在云端建立该学生对应的**科目分类表**，后续所有OCR识别结果自动归入对应科目
- 初始化时引导学生或家长为AI命名（支持任意命名）
  - 限制规则：不允许使用亲属称谓（爸爸、妈妈、爷爷、奶奶、外公、外婆等通用家庭称呼）
  - AI以命名后的名字自称，增强亲切感
- 选择AI的**交互声音**和**讲话节奏**（如：温柔女声/活泼男声，语速快/中/慢）
- 以上配置在使用过程中可随时修改（设置入口常驻）

### 3.2 账号登录功能

- 双端支持**账号密码登录**，一套账号体系
- 登录后保持登录状态**至少24小时**，超过有效期后静默刷新 token，尽量避免要求重新登录
- 同一账号支持一台学生端 + 一台家长端同时在线
- 登录时自动识别设备角色（学生端/家长端）

### 3.3 数据同步

- 双端实时同步核心数据：学习记录、AI交互记录、OCR识别结果、错题集、权限设置
- 所有数据通过服务器中转，依托WebSocket实现实时推送
- 离线暂存，重新联网后自动同步

## 四、学生端功能性需求

### 4.1 语音长聊天模式（登录后默认界面）

- 登录后**直接进入语音聊天界面**，无需额外操作
- 交互方式为**实时语音对话**，类似面对面聊天：
  - 无发送按钮，语音实时采集
  - 根据学生**说话停顿时长**智能判断是否结束发言（如停顿1.5秒以上视为说完，AI开始回答）
  - 界面上实时显示**语音转文字**内容（学生和AI双方的对话文字均显示）
  - AI回答时通过语音播放，同时显示文字
- 对话界面为标准聊天气泡样式，左侧为AI消息，右侧为学生消息
- 语音识别支持中文普通话，适配K12学生的口语表达习惯
- 所有对话记录上传云端存储

### 4.2 智能场景切换

- AI具备**场景感知能力**，通过大模型判断学生意图自动切换界面模式：
  - 学生说类似"我要开始做作业"、"帮我看看这张卷子"、"打开摄像头" → AI切换到**摄像头模式**
  - 学生说"关掉摄像头"、"我做完了"、"我想聊天" → AI切换回**聊天模式**
- 切换时AI语音确认："好的，我帮你打开摄像头，把试卷放在镜头前吧"
- 模式切换平滑过渡，语音对话在任何模式下都保持活跃

### 4.3 摄像头模式

#### 4.3.1 画面采集与对焦引导

- 进入摄像头模式后，浏览器实时显示USB摄像头采集的画面
- 支持选择摄像头（内置/USB外接）
- 画面上显示**对焦引导框**，提醒学生"请将书本或试卷对齐到框内"
- AI通过语音引导对焦："把试卷往左移一点"、"好的，已经对齐了"
- 对焦完成后自动触发OCR识别

#### 4.3.2 左右分屏模式

- 对焦完成后，画面切换为**左右分屏**：
  - **左侧**：USB摄像头实时画面（持续更新）
  - **右侧**：OCR识别结果（结构化显示识别出的题目）
- 当OCR识别到内容后，**右侧成为主画面**（占比更大）
- 语音对话切到后台保持实时活跃，学生随时可以语音提问
- 分屏比例可通过拖拽调整

#### 4.3.3 手指指认交互

- 学生可以**指着书本/试卷上的某道题**，同时通过语音与AI交互：
  - 示例："这道题怎么做？" → AI识别手指指向的题目
  - 示例："帮我批改这道题" → AI批改指向的题目
- AI先确认意图再行动：
  - 学生："这道题怎么做？"
  - AI："你想让我教你解这道题吗？"
  - 学生："是的" → AI开始讲解
- 指认通过摄像头画面中的手指位置与OCR题目区域的映射关系实现
- 手指在目标区域停留约1秒视为确认选中

#### 4.3.4 遮挡检测与提醒

- 在摄像头模式下，AI持续检测画面质量
- 当学生手臂/身体**遮挡了摄像头画面中的题目区域**时，AI主动语音提醒：
  - "你挡住我了，我看不到题目啦"
  - "可以移动一下摄像头，或者换个姿势"
- 遮挡解除后AI确认："好了，现在看得清楚了"

### 4.4 AI批改与讲解

- 批改和讲解通过**语音+视频指认**的方式进行，不需要按钮操作
- 批改流程：指认题目 → AI确认 → 显示批改结果（对/错、正确答案、解析）
- 讲解流程：AI通过语音讲解解题思路，同时在右侧面板高亮对应步骤
- AI讲解内容贴合学生认知水平，语言通俗易懂
- 错因分析自动分类：计算失误、审题不清、概念模糊、公式记错、步骤缺失等

### 4.5 AI个性化配置

- AI有名字，在对话中以自己的名字自称
- 学生或家长可随时修改：
  - AI名称
  - AI声音（多种音色可选）
  - AI讲话节奏（语速快/中/慢）
- 配置入口：聊天界面顶部设置图标，或通过语音命令修改

### 4.6 错题处理

- AI批改后的所有错题，自动收录并上传云端
- 错题**首先推送给家长端**，由家长决定是否同步到学生端
- 如果家长勾选"后续都推送"，则后续错题自动同时推送双端
- 错题包含：原题截图、学生答案、正确答案、错因分析、知识点标签、所属科目/章节

## 五、家长端功能性需求

### 5.1 远程介入教学（核心功能）

- 在学生做作业过程中，家长有权**远程接入**学生端
- 远程接入后的画面交互：
  - 家长画面以**画中画**形式显示在USB摄像头画面的**右上角**
  - 点击家长画面 → 家长画面**放大**顶替USB摄像头主画面，USB画面缩小至**左上角**
  - 再点击USB小画面 → USB画面恢复放大，家长画面缩回**右上角**
  - 两个画面可反复切换
- 家长远程介入后可以：
  - 与AI交流（AI同时理解家长和学生的语音）
  - 与学生交流
  - 将某道题推送给学生端
  - 将生成的试题集推送给学生端
- **家长介入后AI行为变化**：
  - AI自动减少主动发言
  - 除非AI被直接提问（家长或学生问AI），AI才回答
  - AI在家长介入时更多充当辅助角色

### 5.2 OCR识别结果管理

- 所有OCR识别结果**自动上传云端**
- 家长可以：
  - 按**日期**查看任意一天的识别试题
  - 按**科目**筛选查看
  - 查看该日的**错误题目**列表
  - 通过**大模型搜索**试题或错误题目（自然语言搜索，如"上周数学做错的分数题"）

### 5.3 题目自动归类

- 初始化时根据学生年级和就读区域，大模型自动建立**科目分类表**
- OCR识别的所有试题自动归入对应科目和章节
- 归类由大模型智能判断（根据题目内容识别所属科目、知识点、章节）
- 家长可查看完整的科目树结构和每个章节下的题目

### 5.4 错题集创建与试题生成

- 家长可以自主创建错题集：
  - 选择**科目**
  - 选择**时间段**（如：最近一周、本月）
  - 选择科目中的**具体章节**
- AI根据学生做错的试题，通过大模型**自动生成练习试题**
- 试题生成规则：
  - 家长可设置**正确题目与错误题目的比例**（即简单题vs针对错误知识点的题）
  - 默认比例：**正确2 : 错误8**（20%基础巩固题 + 80%针对薄弱点的题）
  - 也可选择"自动生成"，由AI根据学生情况自行决定比例
- **AI自审核机制**：
  - 生成的试题先经过大模型自身审核
  - 审核内容：题目是否符合年级水平、答案是否正确、是否与原错题知识点匹配
  - 审核不通过的题目自动重新生成替换
- 生成的试题集家长可以：
  - 推送给学生端
  - 选择打印

### 5.5 错题推送机制

- AI批改后的错题**首先推送家长端**
- 家长可以：
  - 逐条审核，选择是否推送到学生端
  - 勾选**"后续都推送"**：开启后，后续所有错题自动同时推送双端
- 推送到学生端的错题显示在学生端的错题列表中

### 5.6 权限管控

- AI解答权限：控制学生端是否可使用AI解答、讲解功能
- 权限修改实时同步学生端，立即生效
- 权限修改日志云端存储，可追溯

### 5.7 学习数据查看

- 查看学生完整学习记录：每日学习时长、题目完成量、正确率变化
- 查看所有科目的错题分布情况
- 查看AI交互记录

## 六、AI核心服务功能性需求

### 6.1 语音对话服务

- 实时语音识别（STT）：将学生语音转为文字
- 语音合成（TTS）：将AI回复转为语音播放，支持多种音色和语速
- 对话上下文管理：维持长对话记忆，理解上下文
- 意图识别：判断学生是在闲聊、提问、请求批改、请求切换模式等
- 停顿检测：根据说话停顿时长判断是否发言结束

### 6.2 试题处理服务

- OCR识别：通过DeepSeek Vision识别印刷体和手写体，覆盖K12主流题型
- 自动批改：对照正确答案批改学生作答
- 题目讲解：生成文字+语音讲解，包含解题思路、知识点、易错点
- 错因分析：自动分类错误原因
- 知识点归类：识别题目所属科目、章节、知识点

### 6.3 手指指认识别服务

- 实时检测摄像头画面中的手指位置和指向
- 将指尖坐标映射到OCR识别的题目区域
- 配合语音指令确定学生意图

### 6.4 遮挡检测服务

- 持续监测摄像头画面质量
- 检测到大面积遮挡时触发语音提醒
- 遮挡解除后确认恢复

### 6.5 试题生成与自审核服务

- 根据学生错题和薄弱知识点生成针对性练习题
- 支持按比例混合基础题和薄弱点题目
- 内置审核机制：验证题目质量、答案正确性、难度匹配度
- 审核不通过自动重新生成

### 6.6 智能搜索服务

- 支持自然语言搜索试题和错题（如"上周的数学错题"、"分数相关的题目"）
- 搜索范围覆盖所有云端存储的OCR识别结果

## 七、双端协同核心流程

### 7.1 日常学习流程

1. 学生登录 → 进入语音聊天界面 → 与AI自由对话
2. 学生说"我要做作业" → AI切换到摄像头模式
3. 学生放置试卷 → AI引导对焦 → OCR识别 → 进入左右分屏
4. 学生指着题目提问 → AI确认意图 → 批改/讲解
5. 错题自动收录 → 推送家长端 → 家长决定是否同步学生端

### 7.2 家长介入流程

1. 家长在家长端点击"远程介入"
2. 学生端显示家长画中画（右上角）
3. 家长可与AI和学生三方交流
4. AI自动减少主动发言，转为辅助角色
5. 家长可推送题目或试题集给学生

### 7.3 错题集生成流程

1. 家长选择科目 + 时间段/章节
2. AI根据错题生成练习试题（按设定比例）
3. AI自审核 → 不合格自动替换
4. 家长预览 → 推送学生端 / 打印

## 八、技术选型

| 层级 | 技术 | 说明 |
|---|---|---|
| 学生端 (iPad) | Flutter (PWA/原生) | 跨平台，支持Web和原生APP |
| 家长端 | Flutter (Web/Android) | 与学生端共享业务代码 |
| 后端服务 | Python FastAPI | 高性能异步API框架 |
| 数据库 | PostgreSQL + Redis | 持久化存储 + 缓存/实时推送 |
| 实时通信 | WebSocket | 双端数据同步、实时推送 |
| 视频通话 | WebRTC / 声网Agora SDK | 家长远程介入的音视频传输 |
| 文件存储 | MinIO / 本地存储 | 试题截图、OCR结果等文件存储 |
| AI全能力 | DeepSeek API (含Vision多模态) | OCR+批改+讲解+错因分析+试题生成+意图识别+搜索 |
| 语音识别 (STT) | Web Speech API / 讯飞 | 实时语音转文字 |
| 语音合成 (TTS) | Web Speech API / 讯飞 | 文字转语音播放，多音色支持 |
| 手指检测 | MediaPipe | 手指位置实时检测 |
| 部署方式 | 裸机部署 | systemd管理服务进程，Nginx反向代理 |

## 九、补充说明

- 本文档仅明确功能性需求，性能、并发、兼容性、安全等非功能性需求后续单独补充
- 学生端交互以**语音为主**，界面尽量简洁，减少按钮操作
- 家长端侧重管控与数据查看，操作直观明了
- 所有数据交互均通过服务器中转，确保数据一致性
- AI在不同场景下的行为模式需明确区分（独立辅导 vs 家长介入时的辅助角色）
